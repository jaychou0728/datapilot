# DataPilot Phase 2 — 设计文档

> 日期：2026-07-14
> 状态：已确认

## Phase 2 目标

让 DataPilot 成为可以交付给真实用户使用的产品——用户系统、数据隔离、操作历史、AI 报告、PDF 导出。

## 实施顺序

```
Task 1: Auth 地基 → 2: Dashboard 统计 → 3: 操作历史 → 4: AI 报告 → 5: PDF 导出
```

每个 Task 建完可独立验证，不依赖后续 Task。

```python
# User
```

```
frontend/src/api/auth.ts        # register(), login(), getMe()
```

### 改动文件

```
backend/app/api/datasets.py     # 上传时关联owner_id，查询时按owner_id过滤
backend/app/model/dataset.py    # DatasetInfo新增owner_id
backend/app/main.py             # 注册auth路由 + auth中间件
backend/app/config.py           # 新增JWT_SECRET, DEFAULT_USER_ID

frontend/src/api/client.ts      # axios拦截器自动加Authorization header
frontend/src/router/index.ts    # 路由守卫：未登录跳/login
frontend/src/App.vue            # 顶栏显示用户名+退出按钮
```

### 关键设计决策

- **JWT 不过期**（Phase 2 简化），token 存 localStorage
- **向后兼容**：配置 `DEFAULT_USER_ID`，无 token 请求自动归属默认用户（Phase 1 数据不丢失）
- **密码**：bcrypt 哈希，最小 6 位
- **注册**即登录：注册成功直接返回 token

### API

- `POST /api/auth/register` → `{username, password}` → `{token, user}`
- `POST /api/auth/login` → `{username, password}` → `{token, user}`
- `GET /api/auth/me` → 当前用户信息（需 token）

### API

### 触发点

- datasets.py → upload 成功 → log
- cleaning.py → execute 成功 → log
- query.py → execute 成功 → log
- charts.py → recommend 成功 → log
- (Task 4) reports → generate 成功 → log

### 前端

## Task 4: AI 分析报告

### 流程

```
用户点"生成报告"（按钮在数据详情页）
  → POST /api/reports/generate {dataset_id}
  → 后端：
    1. 读取数据画像（复用chart_service的_profile_data）
    2. 调用DeepSeek分4段生成：
       a. 概览段：数据规模、字段说明、关键统计
       b. 趋势段：时间维度分析 + 折线图echarts配置
       c. 对比段：分类维度对比 + 柱状图echarts配置
       d. 异常段：异常值检测结果 + 建议
    3. 组装为完整报告JSON：{sections: [{type: text|chart, content}], title, dataset_name, generated_at}
    4. 存到 reports/{report_id}.json
    5. 记录操作历史
  → 返回 report JSON
```

### 新增文件

```
backend/app/model/report.py      # Report模型
backend/app/api/reports.py       # generate, list, get, delete
backend/app/service/report_service.py  # 报告生成逻辑

frontend/src/views/ReportDetail.vue   # 报告详情页（文字+可交互图表混排）
frontend/src/api/reports.ts
```

### 前端

- 报告详情页：文字段落 + 内嵌 ChartPanel（可交互的 ECharts），类似 Notion 的图文混排
- 数据详情页新增"生成报告"按钮
- 首页/导航新增"报告"入口

### 路由

- `/reports` → 报告列表
- `/reports/:id` → 报告详情

## Task 5: PDF 导出

### 流程

```
报告详情页点"导出PDF"
  → GET /api/reports/{id}/export
  → 后端：
    1. 读取报告JSON
    2. 用Jinja2渲染HTML模板（报告内容 + 图表静态渲染）
    3. Playwright打开HTML → 截图所有图表 → 替换为图片
    4. 生成PDF（A4，含封面页）
  → 返回 PDF 文件流
```

### 新增文件

```
backend/app/service/pdf_service.py   # HTML→PDF渲染
backend/app/templates/report.html    # 报告HTML模板
```

### 依赖

`pip install playwright weasyprint jinja2`

## 数据库 Schema 汇总

```sql
-- User
CREATE TABLE users (
    id TEXT PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- OperationLog (存SQLite)
CREATE TABLE operation_logs (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    type TEXT NOT NULL,
    detail TEXT NOT NULL,
    dataset_id TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Report (存JSON文件 + SQLite索引)
CREATE TABLE reports (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL,
    dataset_id TEXT NOT NULL,
    title TEXT NOT NULL,
    file_path TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

SQLite 文件位置：`backend/data.db`（新增）
